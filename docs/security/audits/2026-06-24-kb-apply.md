## Fixed live this session (verified by HTTP probe + fleet-heartbeat check, 0 rejected)
- **DCP-API-01**: agent-gateway now gates on provider-key presence (`DC1_GATEWAY_REQUIRE_KEY=1`)
  + 60/min/IP rate limit. Keyless POST -> **401**; keyed path unaffected; `/health` exempt.
  Nexus is NOT a gateway caller (it hits DeepSeek/OpenRouter direct), so no agent breakage.
  **CORRECTION (Peter):** the upstream “Token Plan usage limit reached” the scan observed was MiniMax
  expiring from **non-payment, NOT attacker exhaustion**; the keyless log calls were this audit’s OWN
  verification probes. So this was **never a realized incident** — it is a *latent* hole (the proxy can
  route to a metered upstream e.g. ANTHROPIC_API_KEY → unbounded), which is why gating it is still correct.
  *Phase-2 (staged): validate the key against providers/renters + meter usage like /v1.*
- **DCP-API-03**: `requireAdminAuth` on `recovery.js:/resolve/:event_id` and
  `fallback.js:/simulate` (the two unauth DB-writes). GET reads left open (no dashboard breakage).
  Both writes -> **401** unauth.
- **AI-1**: Nexus `tirith` injection/command scanner RE-ENABLED (it had regressed to `false`
  because `tirith_path: tirith` was not on the hermes user PATH). Set `tirith_path:
  /opt/data/bin/tirith` (verified `tirith 0.3.3` runs as hermes), `tirith_enabled:true`,
  `tirith_fail_open:false`. Nexus restarted clean.
- **AI-3**: Nexus `memory.write_approval:true` — durable memory writes now require approval,
  closing the one-injection-becomes-persistent poisoning path.
- **AI-4**: Spark `tirith_fail_open:false` + correct `tirith_path` — guardrail no longer fails open.
