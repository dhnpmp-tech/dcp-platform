## Provider onboarding checklist

1. Register as provider at `/setup`.
2. Store the API key returned during registration for daemon setup.
3. Install the daemon for your OS.
4. Confirm heartbeat status becomes `online` and remains stable for several check cycles.
5. Keep your machine online to receive scheduled jobs.

## Prerequisites

- NVIDIA GPU (8 GB VRAM minimum, 16+ GB recommended)
- Updated NVIDIA drivers (`nvidia-smi` must work)
- Python 3.8+
- Docker + NVIDIA Container Toolkit (recommended)
- Stable internet connection

## Install daemon (Linux/macOS)

```bash
curl -sL "https://dcp.sa/api/dc1/providers/download/setup?key=<PROVIDER_KEY>&os=linux" | bash
```

## Install daemon (Windows PowerShell)

```powershell
irm "https://dcp.sa/api/dc1/providers/download/setup?key=<PROVIDER_KEY>&os=windows" | iex
```

## Verify provider status

```bash
curl -s "https://dcp.sa/api/dc1/providers/me?key=<PROVIDER_KEY>"
```

Expected fields:

- `provider.status` is `online`
- `provider.last_heartbeat` updates every 30 seconds
- `provider.total_jobs` increments after completed jobs

## Daemon lifecycle

- Heartbeat interval: 30s (`POST /api/providers/heartbeat`)
- Job poll interval: daemon polls provider queue endpoint
- Result reporting: `POST /api/providers/job-result`

## Earnings and settlement

- Job revenue split: provider `75%`, platform `25%`
- Currency: SAR, internal precision in halala
- Use provider wallet and job history in your dashboard for available balance and total earnings.

If you run on Saudi infrastructure, utilization can be more predictable for steady workloads than highly volatile spot-like demand patterns.

## Troubleshooting quick checks

1. `nvidia-smi` returns your GPU model.
2. API key is correct (`dc1-provider-...` or the currently returned provider key prefix).
3. Daemon can reach `https://dcp.sa/api/dc1`.
4. Firewall allows outbound HTTPS.

## Runtime verification (operator baseline)

Before declaring a provider machine production-ready, verify host runtime and API wiring:

```bash
# GPU + driver visibility
nvidia-smi

# Docker sees NVIDIA runtime
docker info | grep -i runtime

# Provider heartbeat + status
curl -s "https://dcp.sa/api/dc1/providers/me?key=<PROVIDER_KEY>"
```

Then run end-to-end smoke validation from submit to output retrieval. Full checklist:
