Brand Architecture

DCP — What the letters mean

D = Decentralised  ·  C = Compute  ·  P = Power

The ∞ symbol carries the brand story: infinite scale, and the perpetual energy→compute cycle. Saudi energy powers compute. Compute powers the digital economy. The loop never stops.

Purpose, Vision & Mission

Purpose — Why we exist

Saudi Arabia has abundant, affordable energy and idle compute capacity. DCP exists to unlock that potential — making world-class AI infrastructure accessible to every builder in the Kingdom and across MENA.

Vision — Where we're going

A world where any developer, startup, or enterprise in Saudi Arabia can access enterprise-grade GPU compute as easily as electricity — priced at Saudi rates, governed by Saudi law.

Mission — What we do daily

Build, operate, and scale a decentralised compute platform that connects idle GPU capacity with AI workloads — compliantly, reliably, and at a price that makes hyperscalers irrelevant in MENA.

Brand Values

🔒 Sovereignty

Data stays in the Kingdom. Your compute workload executes in Saudi Arabia. This is not a feature — it is the foundation everything else is built on.

⚡ Performance

We obsess over compute efficiency, uptime, and speed. Enterprise clients don't tolerate excuses. Neither do we.

🌐 Openness

Decentralised means open. Open architecture, portable workloads, no lock-in. Any provider, any client, one platform.

🤝 Trust

Compliance commitments made are compliance commitments kept. Every contract, every SLA, every uptime guarantee is non-negotiable.

Logo Usage — Do & Don't

✅ Do
  • Use on navy (#0D1B2A) or near-black backgrounds
  • Maintain minimum clear space (= height of "D")
  • Use provided SVG/PNG files — don't recreate
  • Use monochrome version on non-brand backgrounds
  • Scale proportionally only
  • Use dcp-logo-512.png for social avatars (1:1 ratio)
❌ Don't
  • Don't place on light or mid-tone backgrounds without monochrome version
  • Don't change the gradient colours
  • Don't stretch, skew, or rotate the logo
  • Don't add drop shadows or additional effects
  • Don't use at less than 120px wide (30mm print)
  • Don't place wordmark and ∞ symbol separately
  • Don't outline or stroke the letterforms

Brand Voice & Messaging

Voice Characteristics

Direct — say what we mean

No jargon. No "synergize", "leverage", "circle back", or "solutions". We talk to engineers. Engineers hate filler.

Technical — don't dumb it down

Our audience are CTOs, AI developers, infrastructure engineers. Use CUDA, VRAM, PCIe, inference, training run. They know these words — use them accurately.

Confident — own the position

Don't hedge. "Saudi energy rates are our structural moat" — not "we believe we may be able to offer competitive pricing". We know the numbers. Say them.

Saudi-Powered — infrastructure advantage, global audience

We run on Saudi energy economics — $0.048/kWh, 60–70% cheaper than hyperscalers. That advantage is for builders everywhere. Don't narrow the audience to Saudi builders. The platform is global; the power source is Saudi. "Saudi energy rates are your competitive edge" — not "built for Saudi Arabia".

Brief — every word earns its place

Respect reader time. Write the shortest version that's accurate. Then cut 20% more. If it doesn't add information, delete it.

✅ DO — Hero headline
"Rent GPU compute in Saudi Arabia. Your data stays here."
❌ DON'T — Generic
"Enterprise-grade cloud computing solutions for your organisation's diverse needs."

Motion & Animation

🔧 For developers: Motion communicates state. It is never decorative. Fast, purposeful, predictable — Vercel-style. Never slow, bouncy, or attention-seeking.

Core Principles

  • Functional: Motion clarifies state changes. Hover lifts. Loading spins. Transitions slide.
  • Fast: Snappy. Never laggy. If an animation takes >300ms, it had better be a page-level transition.
  • Accessible: Always respect prefers-reduced-motion. Users with vestibular disorders will get nauseous if we don't.
  • Consistent: Same timing for the same type of interaction, everywhere.

Timing Scale

Duration Token Use Case Examples
100ms --dcp-fast (sub) Instant feedback Button active press
150ms --dcp-fast Micro-interactions Button hover, checkbox toggle, menu open
200ms --dcp-normal UI state changes Tab switch, tooltip appear, badge update
300ms --dcp-slow Page transitions Router navigation, sidebar collapse, modal fade
500ms+ --dcp-slower Complex animations Scroll reveals, staggered lists, onboarding

Easing Functions

/* DCP Standard Easing Curves */
--dcp-ease:       cubic-bezier(0.4, 0, 0.2, 1)  /* Standard smoothness (default) */
--dcp-ease-out:   cubic-bezier(0, 0, 0.2, 1)    /* Entrance: element decelerates */
--dcp-ease-in:    cubic-bezier(0.4, 0, 1, 1)    /* Exit: element accelerates */
--dcp-ease-sharp: cubic-bezier(0.4, 0, 0.6, 1)  /* Quick state changes */

Common Animation Specs

Button Hover

Hover: transform: translateY(-1px); box-shadow: var(--dcp-shadow-teal);

Duration: 150ms ease-out

Active press: transform: translateY(0); — 100ms ease-in

Page/Route Transition

Enter: fadeIn + translateY(8px → 0) — 300ms ease-out

Exit: fadeOut + translateY(0 → −8px) — 200ms ease-in

Loading Spinner

Animation: 360° rotation — 1s linear, infinite

Color: Teal #00E5C8

Size: 20px standard, 32px large

Skeleton Loading

Animation: Pulse shimmer — 1.5s ease-in-out, infinite alternate

Colors: --dcp-surface-3 → --dcp-surface-4

Use when: Data load >200ms — skeleton always beats "Loading..." text

Reduced Motion (CRITICAL)

⚠️ This is non-negotiable. Include this in every global stylesheet. Users with vestibular disorders will experience motion sickness without it. 
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration:       0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration:      0.01ms !important;
    scroll-behavior:          auto !important;
  }
}

Focus States

Focus ring: outline: 3px solid #00E5C8; outline-offset: 2px;

Timing: Instantly visible — no delay or fade-in on focus

Required on: All interactive elements — buttons, links, inputs, selects, checkboxes, radio buttons, dropdowns

Never: outline: none without a custom focus visible implementation

Keyboard Navigation

  • All interactive elements keyboard-accessible via Tab
  • Tab order is logical: left-to-right, top-to-bottom (reversed in RTL)
  • Skip navigation link at top of every page
  • Modal focus trap — Tab stays within open modal
  • Escape key dismisses modals and dropdowns
  • Enter/Space activate buttons; Enter submits forms
  • Arrow keys navigate within menus and select components

Screen Reader Requirements

/* Icon-only buttons MUST have aria-label */
<button aria-label="Close modal">
  <svg ...></svg>
</button>

/* Form inputs MUST have associated visible or SR labels */
<label for="gpu-model">GPU Model</label>
<input id="gpu-model" type="text" />

/* Status indicators MUST announce changes */
<div role="status" aria-live="polite">

</div>

/* Errors MUST be associated with their field */
<input aria-describedby="email-error" aria-invalid="true" />
<p id="email-error" role="alert">Enter a valid email address</p>

Colour & Information Encoding

❌ Never convey information by colour alone.
  • ❌ Red background alone = error (must add icon + text)
  • ❌ Green border alone = success (must add checkmark + text)
  • ✅ Always combine: colour + icon + text label

RTL Layout Rules

/* Use CSS logical properties — work in both LTR and RTL */

/* ✅ DO THIS */
.card {
  padding-inline:       16px;   /* Works in both LTR + RTL */
  margin-block:         16px;
  inset-inline-start:   0;      /* = left in LTR, right in RTL */
  border-inline-start:  3px solid var(--dcp-teal);
}

/* ❌ DON'T DO THIS */
.card {
  padding-left:  16px;   /* Breaks in RTL */
  margin-bottom: 16px;   /* Directional only */
  left:          0;      /* Wrong direction in RTL */
  border-left:   3px solid var(--dcp-teal);  /* Wrong side in RTL */
}

/* Mirror directional icons in RTL */
[dir="rtl"] .icon-chevron-right { transform: scaleX(-1); }
[dir="rtl"] .icon-arrow-forward { transform: scaleX(-1); }
[dir="rtl"] .icon-reply         { transform: scaleX(-1); }

RTL Layout Checklist

  • DCP logo/wordmark stays left-to-right (brand mark is universal)
  • Navigation reverses order in RTL view
  • Directional icons are mirrored (chevrons, arrows)
  • Form inputs flow right-to-left for Arabic text
  • Table layout: column order reverses; cell text aligns right
  • Progress bars fill right-to-left
  • All interactive elements function identically in RTL
  • Sidebar appears on right in RTL layout
