§ 03 · What we hold Storage class · retention · jurisdiction
Class A · Prompt

Raw prompt text

Encrypted in-flight (TLS 1.3) and at-rest (AES-256). Held only as long as the request is processing; flushed at session close.

Retention · session 🇸🇦 KSA
Class B · Embeddings

Vector representations

Customer-owned vectors written to your project's KSA index. Persisted on encrypted block storage. You can purge at any time from the console.

Retention · until purged 🇸🇦 KSA
Class C · Generation

Model output text

Streamed to your client. Not retained server-side. Optionally cached in your project's KSA cache if you turn caching on.

Retention · none (default) 🇸🇦 KSA
Class D · Audit log

Request envelope + verifier outcome

JSONL row per request — model, region, token counts, residency tag, verifier pass/fail, stripped-citation count. Used for billing reconciliation and PDPL audits.

Retention · 13 months 🇸🇦 KSA
Class E · Payment

Billing & invoicing

mada, Apple Pay, bank transfer; USDC on Base. Card PANs never touch our servers (tokenised at gateway). VAT-grade invoices retained 7 years per ZATCA.

Retention · 7 years 🇸🇦 KSA
Class F · Frontier (opt-in)

DeepSeek V4 prompt + response

Only when sovereign-only mode is off and the router decides. The request payload is mirrored to the PRC endpoint over TLS 1.3 and never written back to KSA storage. Every call gets a flag in the audit log.

Retention · n/a · cross-border 🌐 Cross-border

§ 05 · Cross-border, by your hand Opt-in · revocable · per-project

Frontier mode is off by default. You turn it on.

When a tenant flips on the frontier tier, every API request that the router decides to send to DeepSeek V4 gets a cross-border marker in the audit log, a separate SAR cost line on the invoice, and an inline warning in console.dcp.sa. Sovereign-only mode pins routing to KSA models forever — no escape hatch, no "just this once."
