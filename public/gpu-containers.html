01 — the box

A blank box, and it's yours.

The whole GPU
Not a sandboxed slice of a card — the entire GPU, exclusively. nvidia-smi shows all of it.
SSH & Jupyter
Every pod is reachable over SSH; the PyTorch image also surfaces a JupyterLab access_url.
Open internet
Pull models from Hugging Face, pip install anything, push your results out.
Isolated & capped
Each pod is sandboxed to its own tenant and resource-capped — memory, processes, and dropped capabilities — so one renter's workload can't reach into another's.
Per-second billing
Set a duration; pay for the GPU-seconds you actually hold. Stop any time.
02 — launch

Three ways in. Same endpoint.

CLI, web, and raw API all drive POST /api/pods.

a · CLI

dcp pod

The dc1 SDK ships a dcp CLI.

pip install dc1
export DCP_API_KEY=dc1-renter-…

dcp pod create \
  --image pytorch \
  --duration 60 \
  --token $(uuidgen)

dcp pod list · get · stop
b · API

raw HTTP

One curl, the x-renter-key header.

curl -X POST \
 api.dcp.sa/api/pods \
 -H "x-renter-key: …" \
 -d '{
   "image":"pytorch",
   "duration_minutes":60
 }'
# → id · ssh_command
#   access_url · root_pw

05 — the mesh

On a real Saudi GPU mesh.

A private WireGuard mesh that pools GPUs across three tiers — and keeps renters running when a node drops.

tier · supply

Three-tier mesh

Consumer cards, DCP's own fleet, and datacenter-grade GPUs — one mesh, residential nodes connect outbound from behind NAT. No port-forwarding, no static IP.

resilience

Failover built in

A liveness monitor marks stale nodes offline; in-flight work is requeued and inference retries other capable providers automatically.

sovereign

Close to home

Saudi/MENA-resident compute — regional, data-resident GPU supply a global marketplace can't trivially stand up.

06 — trust

Built to be trusted.

Reliable where it lands, fair on the meter, isolated from your neighbours.

health-gated

Lands on a live GPU

Your pod only schedules onto a provider the platform has verified healthy — drivers are pinned and every node is health-checked each heartbeat, so you never boot onto a dead card.

fair meter

Server-measured billing

You pay for the GPU-seconds the platform measures server-side, capped to real wall-clock — never a number a node reports about itself.

isolated

Your tenant only

Each pod is sandboxed and resource-capped — memory, processes, and dropped capabilities — so one renter's workload can't reach into another's.
